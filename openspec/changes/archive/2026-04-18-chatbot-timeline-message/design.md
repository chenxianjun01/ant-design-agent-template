## Context

当前聊天界面已经能承接结构化消息，但缺少一种适合展示“连续步骤 / 事件进展”的轻量消息类型。表格和文本都能表达这类信息，但可读性和阶段感都不理想。

## Goals / Non-Goals

**Goals:**

- 新增一个适合流程进展展示的时间轴消息类型
- 保持消息协议和分发器扩展方式一致
- 提供本地 mock 样例，方便立即联调

**Non-Goals:**

- 本轮不引入审批动作按钮
- 本轮不新增复杂交互或回执逻辑
- 本轮不通过 SchemaSlot 渲染时间轴

## Decisions

### 1. 先落地 `timeline`，不先做审批卡片

原因：时间轴消息对现有架构侵入更小，只需要协议、渲染和 mock 三层闭环，不需要先定义审批状态流和动作协议。

### 2. 使用 antd `Timeline` 作为基础渲染

原因：现有依赖已包含 antd，`Timeline` 足够覆盖步骤展示、状态颜色和辅助文案，不需要引入额外 UI 方案。

### 3. 时间轴 schema 先不做强依赖

原因：时间轴消息优先走稳定 content 协议，后续如果需要更复杂 schema 映射，再增量进入 adapter。

## Risks / Trade-offs

- [风险] 时间轴协议过于轻量，后续复杂业务可能仍需审批卡片
  → 本轮定位是补齐流程展示类型，不替代更复杂卡片

- [风险] mock 样例太单一会弱化联调价值
  → 直接提供一个贴近“智能体执行进展”的样例
